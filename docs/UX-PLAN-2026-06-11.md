**التاريخ:** 2026-06-11 · **المنهجية:** فحص متعدد الوكلاء (7 وكلاء) لسبعة أسطح: لوحة التاجر، صفحات التاجر، نواة الإدارة، عمليات الإدارة، نظام التصميم، النصوص والوضوح، بنية المعلومات والتدفقات.

## التقييم العام

| السطح | التقييم | أبرز مشكلة |
|---|---|---|
| لوحة التاجر | 6/10 — مفيدة لكن مزدحمة | ودجت متقدمة فوق الأساسيات؛ خريطة السعودية تكسر الوضع الليلي |

| عمليات الإدارة | متفاوت | 3 طبقات تصميم مختلفة؛ تكرار شاشات المهام |

## المرحلة 1 — مكاسب سريعة عالية الأثر (أيام · جهد منخفض)

أكثر بنود تحسّن الوضوح فوراً بأقل مخاطرة:

### نصوص ومصطلحات
1. **بطاقة AOV**: استبدال «AOV» بـ «متوسط قيمة الطلب» + شرح «متوسط ما ينفقه العميل في الطلب الواحد». (`Dashboard.tsx:505`)
2. **توحيد مبيعات/إيرادات**: عنوان المخطط «اتجاه الإيرادات» ← «اتجاه المبيعات» ليطابق كل الأرقام. كلمة واحدة للمبيعات في كل الواجهة. (`Dashboard.tsx:599`، ABC widget)
3. **تصحيح وصف صافي الربح**: «بعد الرسوم والإعلانات والمرتجعات» بينما المرتجعات **لا تُخصم** فعلياً في الحساب → إمّا تصحيح النص أو إضافة المرتجعات للمعادلة. (`Dashboard.tsx:392,503`)
4. **توحيد «راكد»**: تعني 30 يوم في موضع و90 في آخر على نفس الصفحة → سلّم موحّد: بطيء (30+)، متقادم (60+)، راكد (90+). (`Inventory.tsx:408,488`؛ `Help.tsx`)
5. **مستوى ثقة الذكاء الاصطناعي**: «high» الإنجليزية ← «عالية/متوسطة/منخفضة». (`Dashboard.tsx:791`)
6. **رسائل الأخطاء**: إخفاء نص الخطأ الإنجليزي الخام و«خطأ» المفردة → رسائل ودودة بخطوة تالية. (`Requests.tsx:159`، `Billing.tsx:61`، `Dashboard.tsx:376`)
7. **تسميات الفلاتر**: إضافة «الفترة:» و«المنصة:» قبل صفّي الشرائح في اللوحة. (`Dashboard.tsx:523`)
8. **اقتراحات المساعد الذكي**: «كيف أحسّن الـ ROAS؟» ← «أي إعلاناتي يخسر فلوسي؟». (`AIChat.tsx:11`)
9. **خطوات الإرشاد**: «Dashboard» ← «لوحة التحكم» لتطابق القائمة. (`OnboardingTour.tsx:12`)

### إدارة
10. **مجموعة «التجار» المضلِّلة**: نقل «استيراد ملفات» + «إدخال يدوي» لمجموعة «إدخال البيانات» مثبّتة أعلى القائمة (أكثر شاشة استخداماً). (`AdminPanel.tsx:76-89`)
11. **أيقونات مكررة**: Inbox×2 وTruck×2 في الشريط ← أيقونات مميزة لكل شاشة. (`AdminPanel.tsx:80,81,84,86`)
12. **خرائط الحالات/الباقات المكررة**: «pending» = «قيد المراجعة» هنا و«بانتظار المراجعة» هناك → ملف واحد مشترك. (`adminShared.ts` + كل الشاشات)
13. **DBHealthView**: إخفاؤها خلف صلاحية مطوّر، أو ترجمة «Cache Hit/Webhook» للعربية بشرح. (`DBHealthView.tsx:144`)

## المرحلة 2 — بنية المعلومات والتدفقات (أسبوع · جهد متوسط)

أعلى قيمة لتحقيق «سهل وسلس الوصول للمعلومة»:

1. **بطاقة «أهم 3 إجراءات اليوم»** أعلى اللوحة مباشرة بعد المؤشرات: سطر من كل مصدر موجود — «🔴 أوقف إعلان X (خسر 420 ر.س)»، «📉 منتج Y يبيع بخسارة»، «📦 جدّد مخزون Z (يكفي 5 أيام)» — كل سطر زر ينقل للصفحة. يحوّل ثلاث صفحات مدفونة إلى قائمة مهام واحدة. **(هذا أهم بند في الخطة — يحقق هدف المالك حرفياً)**
2. **بحث ظاهر**: زر بحث في رأس الشريط الجانبي وأيقونة في شريط الجوال (الـ CommandPalette موجود لكنه Cmd+K فقط = خفيّ). (`App.tsx:461`، `CommandPalette.tsx`)
3. **إعادة ترتيب اللوحة** هرمياً: مؤشرات ← اتجاه المبيعات + توزيع المنصات ← أعلى المنتجات/المدن ← ودجت تشغيلية (إعادة التوريد) ← أدوات عميقة (ABC، الخريطة الحرارية، خريطة السعودية) **مطويّة**. (`Dashboard.tsx:567-592`)
4. **إصلاح قائمة الجوال للتاجر**: تُسقط حالياً «التسويق» وكل ما بعد العنصر الخامس → استبدال زر الخروج بتبويب «المزيد» يفتح كل الوجهات. (`App.tsx:487-503`)
5. **دمج صفحات متلازمة** (تبويبات داخل صفحة واحدة، نفس الكود):
   - الطلبات + كشف الحساب ← «المبيعات والمستحقات»
   - منتجاتي + المخزون ← «المنتجات» (تبويب الربحية/تبويب المخزون)
6. **تبسيط الإرشاد**: الخطوة الأخيرة في نافذة الترحيب زر «🚀 ترقية الباقة» (بيع) ← «ابدأ — ارفع أول تقرير». (`OnboardingFlow.tsx:32`)
7. **توحيد «الدعم» و«مركز المساعدة»** المتجاورين الملتبسين ← «المساعدة» بتبويبَي «الأسئلة الشائعة» و«تذاكري». (`App.tsx:74-78`)

## المرحلة 3 — صفحات التاجر: من جدران لوحات إلى تبويبات (أسبوع · جهد متوسط)

1. **صفحة المنتجات**: 8 لوحات تحليلية فوق الكتالوج ← 3 تبويبات (الكتالوج افتراضياً / المخزون / التحليلات). (`Products.tsx:216`)
2. **صفحة الكشف**: 12 لوحة في تمرير واحد ← تبويبات (ملخص / الربح والتوقعات / المرتجعات / المعاملات)، مع تثبيت «الصافي المستحق». (`Statement.tsx:166`)
3. **الخريطة الحرارية**: على اللوحة اعرض سطر الذروة فقط («⭐ أكثر وقت مبيعاتك: الخميس 8م»)، وانقل الشبكة الكاملة لصفحة التحليلات. (`Dashboard.tsx:1027`)
4. **خريطة السعودية**: إصلاح ألوان الوضع الليلي (CSS vars بدل hex ثابت) + جعلها قابلة للطي + تلميح لمسي للجوال — أو إزالتها (قائمة «أعلى المدن» تكفي). (`Dashboard.tsx:104-268`)

## المرحلة 4 — نظام التصميم (أسبوعان · تدريجي لا دفعة واحدة)

النظام الحالي معطّل: لا مكوّنات مشتركة، 2677 نمط مضمّن، 11 خريطة أنماط مكررة.

1. **توحيد الهوية**: لونان للبنفسجي (`--accent:#6c5ce7` مقابل `#7c6bff` المستخدم 114 مرة) → مصدر واحد. نفس الشيء للذهبي والأحمر. (`index.css:17`)
2. **مقاييس موحّدة (Tokens)**: زوايا (5 درجات)، مسافات (شبكة 8px)، خطوط (6 درجات بدل 22 حجماً عشوائياً).
3. **~8 مكوّنات مشتركة** في `UI.tsx`: `Card`، `KPICard`، `Button`، `Input`، `Field`، `Badge`، `SectionTitle`، `Modal` — تقرأ الـ tokens فيأتي الوضع الليلي وRTL مجاناً. الهجرة صفحة-صفحة (تبدأ باللوحة)، والخرائط القديمة تتعايش أثناء الانتقال.
4. **أيقونات احترافية**: استبدال الإيموجي (📈💰🛒) بأيقونات lucide ملوّنة بلون البطاقة — الإيموجي يبدو نموذجاً أولياً لا منتجاً مالياً. (`Dashboard.tsx:503`)
5. **شريط الإدارة الجانبي**: يثبّت ألوان داكنة بـ hex بدل vars → لا يتبع الثيم. (`adminShared.ts:21-73`)
6. **التلميحات (Tooltip)**: hover فقط = مكسورة على الجوال + النص يفيض → جعلها بالنقر + التفاف النص. **هذا يصلح كل تلميحات المصطلحات دفعة واحدة.** (`UI.tsx:30-52`)

## المرحلة 5 — قاموس المصطلحات الموحّد + تنظيف الإدارة (أسبوع)

### قاموس مصطلحات مدمج في مكان الحاجة
كل مصطلح أجنبي يظهر للتاجر مع شرح عربي قصير (tooltip يقرأ من Help كمصدر واحد):

| المصطلح | الشرح العربي المقترح |
|---|---|
| ROAS | العائد على الإنفاق الإعلاني — كم ريال مبيعات مقابل كل ريال إعلان |
| ACOS | تكلفة الإعلان كنسبة من المبيعات |
| CTR | نسبة النقر — كم نقر على إعلانك ممن شاهده |
| AOV | متوسط قيمة الطلب الواحد |
| GMV | إجمالي المبيعات قبل أي خصم |
| FBA/FBN | مخزون وشحن عبر المنصة (أمازون/نون) |
| ASN | الإرسالية التي أرسلتها لمستودع المنصة |
| GRN | ما استلمه المستودع فعلياً |
| QC | فحص الجودة عند الاستلام |
| SKU | الرمز التعريفي لكل منتج |
| Buy Box | صندوق الشراء — الأحقية بالظهور الأول |

(المواضع: `Inventory.tsx:539`، `Marketing.tsx:147,215,307`، `Products.tsx`، `AdsView.tsx:120`)

### تنظيف تكرار الإدارة
1. **حذف AdminRequestsView**: شاشة مكررة على نفس جدول `merchant_requests` (القائمة تسمّيها «طلبات قديمة» = اعتراف بأنها مهجورة) → توجيه «طلبات» إلى TasksBoardView. (`AdminPanel.tsx:81`)
2. **لوحة الفريق تعدّ جدولاً ثالثاً** (`admin_tasks`) يختلف عن لوحة المهام → مصدر واحد. (`TeamDashboardView.tsx:38`)
3. **دمج اللوجستيات**: «الإرساليات والاستلام» + «العمليات والشحن» المتداخلتان ← شاشة بتبويبَي وارد/صادر. (`AdminPanel.tsx:84,86`)
4. **توحيد الحالات الفارغة**: ~10 شاشات تخترع حالة فارغة خاصة بها ← مكوّن `EmptyState` المشترك مع وصف الخطوة التالية.
5. **عناوين الصفحات**: 5 شاشات إدارية تفتح بلا عنوان/شرح ← رأس موحّد. (`RevenueView`، `PerformanceView`، ...)
6. **عرض اسم التاجر** بدل `merchant_code` في الجداول. (`PerformanceView.tsx:145`)
